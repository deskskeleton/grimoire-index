## Conventions

- Group by **fictional universe / franchise / author** when several systems share a setting (Cosmere, D&D, Marvel) so batching can sweep a universe at once.
- One line per system. Format: `name — work / creator — short hint`.
- Sub-disciplines that warrant their own entry get their own line; trivial sub-disciplines get a parenthetical mention.

### Naming convention for systems without a canon name

Many works have a magic system but no in-universe label for it. **Do not invent poetic or evocative names.** Use a deterministic, source-tied form so every reader can map name → work at a glance:

1. **Canon name** if one exists (in-world, in-paratext, or community-standard). Examples: *Allomancy*, *The Force*, *Bending*, *Nen*, *Reality Marbles*.
2. **Generic in-world term + work qualifier** if the world uses a generic word like "magic" or "alchemy". Examples: *Fullmetal Alchemy* (alchemy is the generic in-world term; we qualify with the work). *Discworld witchcraft*.
3. **`<Work Title> magic`** as the default fallback when no canon name and no useful in-world generic exists. Examples: *Forspoken magic*, *Avowed magic*, *Eiyuden Chronicle Rune-Lens*.
4. **`<Work Title> <key mechanic>`** when the mechanic itself is the most identifiable thing. Examples: *Three-Body Problem sophons*, *The Power skein organ*, *Mashle Magic Marks*.

What to avoid:
- Invented atmospheric labels not from the source ("The Bloom", "The Choir", "The Wheel", "The Lacrima"). These look like canon and aren't.
- Names that don't tell you which work they're from ("Resonance", "The Codex", "The Light").
- Conflating an author's brand with one specific system ("Sanderson magic" — pick the system).

## NOTES / HOLDING PEN

- "Magic" in pure SF (Star Trek, Foundation, etc.) — generally skip unless framed as magic-coded (e.g., the Force already covers SF magic).
- Superhero powers without a system (e.g., Superman) — skip; we want **systems**, not individual abilities.
- Worlds where the mechanic is treated as biology (X-Men mutants, MHA Quirks) are included because the world-mechanic is system-shaped.
- Subsumed: many CRPG magic systems are direct D&D adaptations and should be cross-referenced rather than separately catalogued (e.g., Baldur's Gate 3 = D&D 5e). We'll treat them as references to D&D unless they materially diverge.

## Open questions for batching

1. Treat sub-disciplines as their own entries from the start (Surgebinding split into ten Surges? Probably no), or collapse to one entry per recognizable system?
2. For franchise umbrellas (Cosmere, D&D family, Marvel, Riordanverse), do we batch per-franchise or by mechanic-type?
3. Threshold for "system" vs "individual ability": the rule of thumb is **rules that apply to multiple practitioners with predictable outcomes**. Single-character bespoke powers are out.
4. Korean/Chinese web novels: how much depth? They're enormously popular but English-language coverage is uneven.
